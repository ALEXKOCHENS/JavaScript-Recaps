Let's Start Simple: What Are We Talking About?

Okay friends, When you use any app on your phone, what do you see? Screens, right? A login screen, then maybe a home screen, then a details screen, and so on.

In Android, each of these screens is called an Activity. Think of an Activity as a single window where your app does something. If your app was a book, each Activity would be a page.

Now, how do you go from one page to another? You turn the page! In Android, we "turn pages" using something called an Intent.

Simple Analogy Time!

Imagine you're in a shopping mall:

  • The Mall = Your Android App
  • Each Shop = An Activity (LoginActivity, HomeActivity, ProfileActivity)
  • The Mall Directory/Maps = Android System
  • You saying "I want to go to the food court" = Creating an Intent
  • Actually walking to the food court = Launching the Activity

So an Intent is basically you telling Android: "Hey, I want to go to this specific place (Activity) in my app!"

What Exactly Is an Activity?

An Activity is one of the four main components of an Android app (the others are Services, Broadcast Receivers, and Content Providers).

Here's what makes an Activity special:

1 It's a single focused thing the user can do

Like typing a message, viewing a photo, or scrolling through a list.

2 It has a user interface (UI)

Every Activity has a layout file (XML) that defines what buttons, text, images appear on screen.

3 It has a lifecycle

It gets created, started, paused, stopped, and destroyed. Like a plant growing, flowering, and eventually... well, you get the idea.

4 It can be launched from other Activities

This is where Intents come in - they're the "invitations" to start new Activities.

Remember This!

Every Android app must have at least one Activity. When you tap an app icon, Android launches that app's "main" Activity (usually called MainActivity). It's like the front door of a house - the first thing you see when you enter.

How Do We Create an Activity? (Step-by-Step)

Creating an Activity is like baking a cake - you need the right ingredients and steps:

Step 1: Create the Java/Kotlin Class

// This is MainActivity.java - our first Activity
public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main); // This connects to our XML layout
    }
}

Step 2: Create the Layout (XML File)

<!-- This is activity_main.xml - defines what the screen looks like -->
<?xml version="1.0" encoding="utf-8"?>
<LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <!-- A simple button -->
    <Button
        android:id="@+id/myButton"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Click Me!" />

</LinearLayout>

Step 3: Declare the Activity in AndroidManifest.xml

This is like registering your Activity with Android so it knows it exists:

<!-- Inside AndroidManifest.xml -->
<manifest ...>
    <application ...>
        <!-- This declares our Activity -->
        <activity android:name=".MainActivity">
            <intent-filter>
                <!-- This makes it launch when app starts -->
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>
Key Point to Remember

If you create an Activity but forget to declare it in AndroidManifest.xml, your app will crash when you try to launch it. It's like building a room in your house but forgetting to put a door - nobody can get in!

Now, What About Intents?

An Intent is a messaging object you can use to request an action from another app component (like starting an Activity).

Another Analogy - Restaurant Style!

Think of an Intent like an order at a restaurant:

  • You (the customer) = Your current Activity
  • The menu = All possible Activities in your app
  • Your order = The Intent ("I want the burger page")
  • The waiter = Android System
  • The kitchen preparing your burger = The new Activity being created
  • Serving your burger = The new Activity appearing on screen

Two Types of Intents:

Explicit Intents Implicit Intents
What: You specifically name which Activity to start

When to use: When you know exactly which Activity in YOUR app you want

Example: "Open the ProfileActivity in my app" 		What: You describe an action, and Android finds which app can do it

When to use: When you want to use other apps' capabilities

Example: "Open a web page" (could use Chrome, Firefox, etc.)

Putting It Together: Launching Activities

Now comes the fun part! Let's see how we actually launch a new Activity from our current one.

Real Example: Login Screen ‚Üí Home Screen

Imagine we have a login screen (LoginActivity). After the user logs in successfully, we want to take them to the home screen (HomeActivity).

Step 1: Create the Intent

// Inside LoginActivity.java, after login is successful
Intent intent = new Intent(this, HomeActivity.class);

This says: "Create an Intent to go from 'here' (LoginActivity) to HomeActivity."

Step 2: Add Extra Data (Optional)

What if we want to send the username to the HomeActivity?

// Add data to the Intent (like putting a note in an envelope)
intent.putExtra("USERNAME", "john_doe");
intent.putExtra("EMAIL", "john@example.com");

Step 3: Launch the Activity

// This actually starts the new Activity
startActivity(intent);

Step 4: Receive Data in HomeActivity

// Inside HomeActivity's onCreate() method
String username = getIntent().getStringExtra("USERNAME");
String email = getIntent().getStringExtra("EMAIL");

// Now we can use these values
TextView welcomeText = findViewById(R.id.welcomeText);
welcomeText.setText("Welcome, " + username + "!");
Pro Tip!

Always use constants for your extra keys (like public static final String KEY_USERNAME = "USERNAME") instead of hardcoding strings. This prevents typos and makes your code easier to maintain!

Let Me Paint the Complete Picture

Think of your Android app like a TV series:

TV Series Analogy
  • Each Episode = An Activity
  • The "Previously on..." recap = The data you pass through Intents
  • The "Next episode" button = The startActivity() call
  • The TV remote = The Intent object
  • The TV network = The Android System

When you press "Next episode" (startActivity), you're using the remote (Intent) to tell the network (Android) which episode (Activity) to play next. Sometimes you even include what happened previously (extra data) so the next episode makes sense!

What Actually Happens Behind the Scenes?

  1. You call startActivity(intent) in your current Activity
  2. Android pauses your current Activity (calls onPause())
  3. Android creates the new Activity instance
  4. The new Activity goes through its lifecycle: onCreate() ‚Üí onStart() ‚Üí onResume()
  5. The new Activity appears on screen
  6. Your old Activity might be stopped (onStop()) if it's completely covered

All of this happens super fast, giving users that smooth "screen transition" feeling we're all used to in apps.

Common Mistakes to Avoid

1 Forgetting to declare Activities in AndroidManifest.xml

This is the #1 reason for crashes when launching Activities. Always check your manifest!

2 Passing too much data through Intents

Intents have size limits! If you need to pass lots of data, consider using a database, ViewModel, or passing an ID instead.

3 Using implicit Intents without a fallback

If you use an implicit Intent to open a web page but the user has no browser installed, your app will crash. Always check if there's an app to handle your Intent!

4 Not handling the back button properly

When users press back, they expect to go to the previous screen. Make sure your Activity stack makes sense!
